1. text (Inverted): Разбивает строку на слова (токены). 
   1. Лучший для поиска «содержит слово А и слово Б». Умеет ранжировать (через функцию score()), если использовать специальные движки, но в обычном MergeTree он просто дает сверхбыстрый фильтр.
2. vector_similarity: Это база для следующего этапа (RAG). Позволяет искать «по смыслу» (косинусное сходство). Работает с эмбеддингами от LLM.
3. bloom_filter / tokenbf_v1: Старое поколение. Вероятностные индексы. Могут давать «ложноположительные» срабатывания. Хуже чем text.
4. ngrambf_v1 / sparse_grams: Поиск по подстрокам (не по словам, а по частям слов). Нужно, если вы ищете внутри слов (например, *ино* внутри виноград).
